# Coding Preferences for wiki-graph Project

## File Organization

### Data Files

**Rule**: All data files should be stored in a `data/` subfolder of the project, if one exists.

This includes:
- CSV, TSV, JSON, RDS, RData files
- Shapefiles and spatial data
- Downloaded reference data
- Intermediate processing results
- Exported analysis outputs

**Example directory structure**:
```
wiki-graph/
├── data/
│   ├── crosswalk_ine_igm.rds
│   ├── macrolanguages_final.rds
│   ├── census_table_municipios_2012u.rds
│   ├── gadm41_BOL_3.gpkg
│   └── [other data files]
├── documentation/
│   ├── MACROLANGUAGES.md
│   ├── WIKI_LANGUAGE_DIVERSITY_MACROLANGUAGE_ANALYSIS.md
│   └── [documentation files]
├── src/
│   └── [R/Quarto source files]
└── [R/Quarto analysis scripts]
```

### File Path References

**Always use `here::here()`** for robust file addressing that works regardless of working directory changes or file moves.

**Pattern**:
```r
library(here)

# Reading data
df <- readRDS(here::here("data", "filename.rds"))
df <- read_csv(here::here("data", "filename.csv"))

# Writing data
saveRDS(df, here::here("data", "output.rds"))
write.csv(df, here::here("data", "output.csv"), row.names = FALSE)

# Referencing other files
source(here::here("src", "helper_functions.R"))
```

**Benefits**:
- ✓ Works regardless of working directory
- ✓ Survives file moves/reorganization
- ✓ Self-documenting path structure
- ✓ Cross-platform (Windows, Mac, Linux)

**Example - Bad**:
```r
# ✗ Brittle: fails if working directory changes
df <- read_csv("data/myfile.csv")

# ✗ Platform-dependent
df <- read_csv("C:/Users/bjorkjcr/Dropbox (Personal)/R/wiki-graph/data/myfile.csv")

# ✗ Relative to script location (confusing)
df <- read_csv("../../data/myfile.csv")
```

**Example - Good**:
```r
# ✓ Always works
library(here)
df <- read_csv(here::here("data", "myfile.csv"))
```

## Code Style

- Use base R pipe `|>` (not magrittr `%>%`)
- Load `library(here)` at the top of scripts that reference data files
- Add comment explaining what data files are used and where they're stored
- For large or frequently-used datasets, note their purpose at the top of the script

## Documentation

- Create `documentation/` files for complex analyses or data transformations
- Use markdown (`.md`) for documentation
- Reference documentation files in code comments:
  ```r
  # See documentation/ANALYSIS_NAME.md for full explanation
  ```
